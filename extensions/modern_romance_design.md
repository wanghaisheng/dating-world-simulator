# 现代都市恋爱模拟系统设计 (Modern Urban Romance Simulation Design)

## 1. 设定概述 (Concept Overview)

本设计将原有的“修仙世界模拟器”架构迁移至**现代都市背景**，核心玩法从单纯的“修仙证道”转变为**“高风险的情感博弈与恋爱攻略”**。玩家扮演一名现代都市中的男性或女性主角，在渴望真爱的同时，必须时刻警惕潜伏在亲密关系中的**猎手与陷阱**。

### 1.1 核心体验
*   **深度情感博弈 (Emotional Gambling)**: 恋爱不再是单向的攻略，而是一场关于金钱、精力与心智的较量。你需要识别真心与假意，在获得“爱的滋养”与遭遇“情感诈骗/PUA”之间如履薄冰。
*   **真实社交模拟**: 基于 AI 生成的真实聊天记录（微信/短信）、朋友圈互动。NPC 会撒谎、会伪装、甚至会通过“煤气灯效应”操控玩家。
*   **多样的恋爱生态**: 利用 LLM 生成极具真实感的 NPC，从提供情绪价值的“完美伴侣”，到索取无度的“吸血鬼”，涵盖高冷总裁、元气偶像、捞女/软饭男、NPD（自恋型人格）等多种原型。
*   **修罗场与危机 (Crisis & Conflict)**: 除了多线操作带来的“翻车”风险（修罗场），玩家还需面对更为致命的“情感破产”——被骗财骗色、精神崩溃甚至社会性死亡。

## 3. 核心机制设计 (Core Mechanics)

### 3.1 角色系统 (Character System)

#### 3.1.1 主角 (The Protagonist)
玩家创建角色时设定：
*   **基础属性**: 外貌、财力、学历、职业。
*   **精力值 (Energy)**: 每日行动点数（上班消耗，约会消耗，睡觉恢复）。
*   **时间管理**: 必须在工作（赚钱）和恋爱（花钱/花时间）之间取得平衡。

#### 3.1.2 恋爱对象 (Love Interests)
NPC 不再是单纯的数据堆砌，而是拥有完整**“恋爱模型”**的智能体。根据主角性别不同，潜在的恋爱对象也会呈现出不同的**阴暗面**与挑战：

*   **通用属性**:
    *   **好感度 (Affection)**: 0-100，决定关系阶段（陌生 -> 熟人 -> 暧昧 -> 恋人 -> 伴侣）。
    *   **依赖度 (Dependency)**: 决定 NPC 对主角的粘人程度。
    *   **真实度 (Sincerity)**: *新增*，隐藏属性。决定 NPC 是真心待你还是另有所图。
    *   **嫉妒值 (Jealousy)**: 当 NPC 察觉主角与其他异性互动时增加。

*   **男性主角的挑战 (Male MC Scenarios)**:
    *   **捞女 (The Gold Digger)**: 提供极高的情绪价值（撒娇、崇拜），但频繁索要昂贵礼物。一旦停止投入，关系迅速降温。
    *   **NPD (自恋型人格)**: 初期进行“轰炸式示爱”，确立关系后开始通过贬低、打压（Gaslighting）来控制主角，榨取情感能量。
    *   **情感诈骗 (The Scammer)**: 伪装成完美伴侣（如“杀猪盘”），诱导主角进行高风险投资或转账，得手后失联。
    *   **扶弟魔/吸血鬼**: 背后有庞大的原生家庭负担，不断通过情感绑架索取资源。

*   **女性主角的挑战 (Female MC Scenarios)**:
    *   **海王 (The Player)**: 同时与多人保持暧昧，擅长提供浪漫体验但拒绝承诺，被发现后通过“假性分手”或“痛哭流涕”挽回。
    *   **PUA 操控者 (The Manipulator)**: 通过建立契约、摧毁自信、切断社交等手段控制主角，将主角变成附属品。
    *   **软饭男 (The Moocher)**: 以怀才不遇为借口，依靠主角的经济支持生活，同时在精神上依赖或剥削主角。

#### 3.1.3 情感博弈：亏损与滋养 (Emotional Dynamics: Loss & Nourishment)
恋爱不仅仅是甜蜜的互动，更是一场关于自我成长的博弈。

*   **爱的滋养 (Nourishment)**:
    *   **情绪价值**: 真正爱你的 NPC 会在主角低谷时提供安慰（恢复大量精力/心情），在事业上提供实质帮助（人脉、资金）。
    *   **共同成长**: 高质量的伴侣会触发“共同进修”事件，提升主角属性（如魅力、才华）。

*   **吃的亏 (Suffering/Loss)**:
    *   **精神内耗**: 与 NPD 或回避型依恋对象相处，会持续扣除心情值，甚至导致“抑郁”状态（无法工作/行动）。
    *   **财富流失**: 被捞女或诈骗犯洗劫，可能导致破产，触发“流落街头”结局。
    *   **自我怀疑**: 长期遭受 PUA 会导致主角“魅力”和“自信”属性永久下降。

### 3.2 互动系统 (Interaction System)

#### 3.2.1 通讯 (Communication)
*   **即时通讯**: 模拟微信/Line 界面。玩家输入或选择回复，AI 生成 NPC 的回复（包含表情包、语音条）。
*   **朋友圈/动态**: NPC 会发布生活动态，玩家的点赞和评论会影响好感度。
*   **深夜夜聊**: 特定时间段（22:00-02:00）的聊天有额外好感加成，但消耗更多精力。

#### 3.2.2 约会 (Dating)
*   **邀约**: 主动发起或被动接受。
*   **场景选择**: 电影院（提升共同话题）、米其林餐厅（消耗金钱，提升虚荣心/浪漫度）、公园（低成本，适合纯爱）、酒店（高亲密需求）。
*   **约会事件**: 约会过程中会触发随机事件（如遇到熟人、钱包丢了、对方前任出现），需要玩家进行选择。

#### 3.2.3 邂逅与日程 (Encounter & Schedule)
*   系统模拟 NPC 的每日日程（上班、健身、逛街）。
*   玩家前往特定地点（如在 NPC 常去的咖啡厅办公）可能触发**偶遇**。

#### 3.2.4 社交软件探险 (Social App Exploration - The Dungeon)
这是现代模式下的“副本系统”。玩家不再去秘境打怪，而是通过社交软件（如“探探”、“Soul”）探索未知的异性资源。

*   **滑卡 (Swipe/Explore)**:
    *   **机制**: 消耗精力 (Energy) 进行“探索”。
    *   **匹配逻辑**: 基于玩家的 **展示面 (Avatar)**（照片、简介、朋友圈）和 **硬价值 (Assets/Career)** 进行概率匹配。
*   **遭遇 (Encounter)**: 每次匹配成功相当于遇到一个“野怪”。
    *   **普通怪**: 普通路人，属性平平。
    *   **精英怪**: 高分男神/女神，攻略难度高，但收益大。
    *   **陷阱怪 (Mimic)**:
        *   **照骗**: 见面后发现颜值 -30。
        *   **酒托/饭托**: 第一次约会就在高消费场所，骗完钱就拉黑。
        *   **杀猪盘机器人**: 诱导投资，榨干资产。
*   **破冰战斗 (Ice-breaking Battle)**:
    *   匹配后的前三句对话至关重要。
    *   系统提供“开场白”选项（幽默/真诚/土味情话）。
    *   **胜利条件**: 对方回复且兴趣值 (Interest) > 0 -> 获得联系方式 (Add Friend)。
    *   **失败条件**: 对方已读不回 (Ghosting) 或 直接拉黑。

### 3.3 恋爱策略与多线机制 (Dating Strategy & Multi-threading)
玩家可以自由选择恋爱策略，系统不进行硬性道德约束，但会根据**“关系深度”**和**“时间重叠”**进行动态判定。

*   **策略选择**:
    *   **纯爱战士 (One by One)**: 每次只攻略一名角色，确认关系前不与其他异性暧昧。风险最低，容易获得“真爱”结局。
    *   **海王/时间管理大师 (The Harem King/Queen)**: 同时攻略多名角色，利用时间差进行多线操作。高风险高收益（情绪价值/物质获取最大化）。

*   **多线重叠规则 (Overlap Rules)**:
    *   **安全区 (Safe Zone)**:
        *   **阶段**: 陌生人 (Stranger) -> 熟人 (Acquaintance) -> 朋友 (Friend)。
        *   **规则**: 在此阶段，玩家可以同时与任意数量的 NPC 保持联系、吃饭、看电影。这被视为正常的社交筛选，不会触发吃醋或修罗场。
    *   **危险区 (Danger Zone)**:
        *   **阶段**: 暧昧 (Crush) -> 恋人 (Lover) -> 伴侣 (Partner)。
        *   **规则**: 一旦进入暧昧期（Affection > 60 或触发特定事件），NPC 会产生**排他性 (Exclusivity)**。此时若被发现与其他异性有亲密互动（如深夜聊天、去酒店、收受贵重礼物），将大幅增加**嫉妒值**，甚至直接触发分手/报复事件。

## 4. AI 叙事与冲突 (AI Narrative & Conflict)

### 4.1 动态聊天引擎
Prompt 重点调整：
*   **风格化**: 严格限制 AI 的说话风格（如傲娇、温柔、毒舌）。
*   **记忆回溯**: 聊天时 AI 会主动提及之前的共同经历（“上次带我去的那家餐厅很好吃...”）。
*   **潜台词**: 高情商 NPC 会说反话，需要玩家识别（如“我没事”可能意味着“快来哄我”）。

### 4.2 修罗场引擎 (The Conflict Engine)
这是本设定的高潮部分。
*   **撞车事件**: 玩家带 A 约会时，偶遇了 B。
*   **查岗**: 敏感多疑的 NPC 会突击检查手机或询问行踪。
*   **多线操作难度**: 随着确立关系的 NPC 数量增加，管理难度呈指数级上升。
*   **结局分支**:
    *   **诚哥结局**: 处理失败，被所有 NPC 抛弃或报复。
    *   **时间管理大师**: 完美平衡，维持多段关系（极高难度）。
    *   **纯爱战神**: 弱水三千只取一瓢，达成完美单人结局。

## 5. 经济与资源 (Economy & Resources)

*   **金钱 (Money)**:
    *   来源：工作（消耗精力）、理财、软饭（从高收入 NPC 处获得礼物）。
    *   用途：购买礼物、支付约会账单、提升自我（整容、健身、进修）。
*   **精力 (Energy)**:
    *   限制玩家每日行动次数，迫使玩家做出取舍（陪 A 还是陪 B？还是加班赚钱？）。

## 6. 技术实现调整 (Technical Implementation)

### 6.1 数据结构 (Data Structures)

#### 6.1.1 现代属性 (ModernProfileMixin)
```python
@dataclass
class ModernProfileMixin:
    # --- 基础属性 ---
    occupation: str          # 职业 (决定收入和接触圈层)
    education: str           # 学历
    mbti: str               # 性格类型 (影响交互偏好)
    hobbies: List[str]      # 兴趣标签
    fashion_style: str      # 穿衣风格
    
    # --- 动态状态 (每日变化) ---
    energy: int = 100        # 精力值 (0-100), 行动消耗, 睡觉恢复
    stress: int = 0          # 压力值 (0-100), 工作/修罗场增加, 娱乐减少
    mood: int = 50           # 心情值 (0-100), 影响交互成功率
    assets: int = 0          # 资产 (现金流)
    
    # --- 隐藏性格属性 (Hidden Traits - 仅对 NPC 有效) ---
    sincerity: int = 80      # 真实度 (0-100). <40 易触发背叛/欺骗
    hidden_archetype: Optional[str] = None # 隐藏原型: GOLD_DIGGER, NPD, SCAMMER, PLAYER, PUA
    
    # --- 恋爱状态 ---
    relationship_status: str # 单身/恋爱中/复杂
    relationships: Dict[str, dict] # 记录与每个人的具体状态
```

#### 6.1.2 关系阶段 (Relationship Stages)
定义关系发展的常数阈值：
```python
class RelationshipStage(Enum):
    STRANGER = 0      # 陌生人 (Affection < 10)
    ACQUAINTANCE = 1  # 熟人 (10 <= Affection < 30) - 安全区
    FRIEND = 2        # 朋友 (30 <= Affection < 60) - 安全区
    CRUSH = 3         # 暧昧 (60 <= Affection < 80) - 危险区 (开始排他)
    LOVER = 4         # 恋人 (80 <= Affection, 确立关系) - 危险区 (严格排他)
    PARTNER = 5       # 伴侣 (订婚/同居)
    EX = -1           # 前任
```

### 6.2 模拟循环 (Simulation Loop)

*   **时间刻度 (Tick)**: 1 Tick = 1 小时。每天 24 Ticks。
*   **日程生成 (Schedule)**: 每日 0 点生成当日日程。
    *   `Work`: 9:00 - 18:00 (根据职业变动)
    *   `Rest`: 23:00 - 7:00
    *   `Free Time`: 其他时间，用于社交/娱乐。

### 6.3 风险引擎 (Risk Engine)

`RelationshipRiskEngine` 是核心逻辑组件，负责处理“负面反馈”和“多线惩罚”。

#### 6.3.1 每日检测 (Daily Check)
针对每一位有过互动的 NPC，系统每日进行一次风险判定：
*   **输入**: 主角状态, NPC 属性 (Sincerity, Archetype), 当前关系阶段, 近期互动记录。
*   **输出**: 触发的风险事件 (Event)。

#### 6.3.2 典型风险事件
1.  **捞女索取 (Gold Digger Demand)**:
    *   *触发条件*: Archetype=GOLD_DIGGER, 关系进入 CRUSH 或 LOVER。
    *   *效果*: 索要高价值礼物。拒绝 -> 好感大幅下降; 同意 -> 资产减少。
2.  **NPD 打压 (NPD Devaluation)**:
    *   *触发条件*: Archetype=NPD, 关系 > CRUSH, 随机触发。
    *   *效果*: 主角 Mood -20, Stress +10。
3.  **多线修罗场 (Harem Conflict)**:
    *   *触发条件*: 在 **危险区 (CRUSH/LOVER)** 阶段，被 NPC 发现与其他异性有亲密行为 (Date/Intimacy)。
    *   *判定公式*: `DiscoveryChance = BaseChance + (IntimacyLevel * 0.1) - (MC_Intelligence * 0.05)`
    *   *效果*: 双方好感清零 / 触发分手 / 触发报复事件 (社会性死亡)。

### 6.4 行动系统 (Action System)

定义具体的玩家行动指令：

| 行动 (Action) | 消耗 (Cost) | 效果 (Effect) | 风险 (Risk) |
| :--- | :--- | :--- | :--- |
| **Work** (工作) | Energy -30, Time 8h | Money ++, Stress + | 偶遇同事 |
| **Chat** (聊天) | Energy -5, Time 0.5h | Affection +, Mood + | 被其他人看到 (低风险) |
| **Date** (约会) | Energy -20, Money -, Time 3h | Affection +++, Mood ++ | **高风险**: 极易被熟人撞见 |
| **Gift** (送礼) | Money -- | Affection ++ | 甚至可能被嫌弃 (捞女) |
| **Snoop** (查岗) | Energy -10 | 发现 NPC 秘密 (如出轨) | 被发现后好感大降 |
| **Swipe** (滑卡) | Energy -15 | 匹配陌生人 (进入副本) | 遇到杀猪盘/照骗 |

## 7. 总结
本设计通过将修仙元素替换为现代都市元素，构建了一个极具张力的恋爱模拟器。AI 的核心价值在于生成**不可预测的恋爱展开**和**真实的情感反馈**，让玩家在虚拟世界中体验极致的甜蜜与修罗场的刺激。
